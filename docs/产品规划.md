# Clipstrate 产品规划

> 一句话定位：**macOS 26 原生玻璃质感、Maccy 级轻量、带"大爆炸"分词拆选的剪贴板管理器。**
> Chop（拆）+ Clip（剪贴板）。前置调研见 [调研报告](调研报告.md)。
> **执行级规格见 [specs/](specs/README.md)（设计已定稿，与本文冲突时以 specs 为准）；UI 参考 `prototype/index.html`。**

## 1. 设计原则

1. **轻且快是硬约束**：常驻内存 < 30 MB、面板唤出 < 100 ms、零内存泄露，全部量化并有自动化验收（见 §5 性能与内存工程）；无 Electron、无常驻 WebView、无网络请求。
2. **原生优先，但不堵死向下兼容**：首发面向 macOS 26 + Apple Silicon（arm64），全量使用 `glassEffect` / `GlassEffectContainer` 等最新 API，不自绘仿玻璃；所有 26-only API 收敛到少数"兼容接缝"（见 §4），日后要支持 macOS 15/16 时只改接缝，不动业务代码。
3. **键盘优先，鼠标同样一等**：所有高频路径既有纯键盘流，也有纯鼠标流。
4. **分词是入口不是目的**：用户要的是"把淘口令/手机号/邮箱一步拿走"，实体识别一键复制优先于精确分词。
5. **本地与隐私**：数据全本地 SQLite；尊重密码管理器的 Concealed 标记；剪贴板隐私弹窗在 onboarding 首屏解决。

## 2. 功能范围

### P0（MVP，必须有）
- 剪贴板历史：文本 / 富文本 / 图片 / 文件路径；`changeCount` 轮询采集；SQLite 存储 + FTS 搜索；图片落盘存缩略图。
- 主面板：全局快捷键（默认 `⌥V`）唤出玻璃浮动面板，列表懒加载、输入即搜、回车粘贴到当前 App。
- **拆词面板（核心差异化）**：三入口 + 词块选取复制，详见 §3。
- 实体识别：邮箱 / 手机号 / URL / 淘口令 / 验证码 / 快递单号。
- Onboarding：引导"剪贴板始终允许" + "辅助功能"两个权限。

### P1（第一次迭代）
- 置顶/收藏、按 App 来源过滤与图标展示、粘贴为纯文本、忽略名单（指定 App 不采集）。
- 实体扩充：身份证、银行卡（Luhn）、IP/MAC、UUID/SHA、金额、日期。
- 拆词面板粒度切换（字 / 词 / 智能短语）。
- 数据导入 / 导出（单文件 `.clipstrate`：配置 + 忽略名单 + 历史库，见 specs 01 §7）。
- 剪贴板堆栈、划词直拆（⌥X 对任意 App 选中文本）。

### P2（观察反馈再做）
- 快捷短语（对标 iCopy 的快速回复）、iCloud 同步、多设备、规则自定义（用户自增正则实体）。

### 明确不做
- 账号体系、云端存储、遥测统计、AI 云端分词。

## 3. 拆词交互设计（核心）

### 3.1 三个入口

| 入口 | 触发 | 场景 |
|---|---|---|
| **A. 复制即拆** | 复制后若检测到实体，屏幕角落浮出胶囊 HUD（2.5s 自动消失），点击或按 `⌥X` 展开拆词面板 | "刚复制了一段带淘口令的分享文案" —— 最高频场景，零额外操作成本 |
| **B. 历史项拆词** | 主面板中任意条目按 `Tab`（或 hover 出现 ✂️ 按钮） | "昨天复制的那条里有个手机号" |
| **C. 划词直拆** | 在任意 App 选中文本按全局快捷键 `⌥X`（AX API 取词，失败回退模拟 ⌘C 并恢复剪贴板现场） | 不想先复制整段，直接对网页/聊天记录选中文本开拆 |

### 3.2 面板形态

非激活浮动 NSPanel（不抢当前 App 焦点），出现在鼠标附近，`GlassEffectContainer` 包裹，词块从原文位置"炸开"落位（liquid glass morphing 过渡，一次性 300ms，可在设置关闭动画）。

```
╭──────────────────────────────────────────────────╮
│  智能识别                                          │
│  ①📱 138****8888   ②📧 user@example.com   ③🛒 ￥AbC…￥(淘口令) │   ← 实体行：按数字键一键复制
│  ──────────────────────────────────────────────  │
│  [复制] [这段] [文案] [，] [快] [打开] [淘宝]        │   ← 词块区：保持原文顺序流式排布
│  [领取] [优惠券] [￥AbCdEf12345￥] [！]              │      标点弱化显示，实体块带图标高亮
│  ──────────────────────────────────────────────  │
│  已选 2 词 · ⏎ 复制  ⇧⏎ 复制并粘贴  空格=连接符 ✕   │   ← 状态栏
╰──────────────────────────────────────────────────╯
```

### 3.3 选取交互（鼠标流）

- **单击**词块：选中/取消，块体玻璃加深 + tint 高亮。
- **按住划过**（刷选）：一笔选中连续多词——这是比 Big Bang 逐点更快的主路径。
- **双击**：整句选中（扩展到句边界）。
- **单击实体块**：直接复制该实体并给出反馈（实体块的语义就是"我要这个"，不需要二段确认）。这是"复制淘口令"的最短路径：复制文案 → HUD 弹出 → 点一下 → 完成，全程一次点击。
- 选中多词时按**原文顺序**拼接（无论点选顺序），默认无连接符，状态栏可切 空格/换行。

### 3.4 选取交互（键盘流）

- `1–9`：复制对应智能实体，面板关闭。**"复制淘口令"全程 = ⌘C → ⌥X → 1，约 1 秒。**
- `←→↑↓`：词块间移动焦点；`空格`：选中/取消；`⇧+方向`：连续刷选。
- `⌘A` 全选；`⏎` 复制所选并关闭；`⇧⏎` 复制并直接粘贴回当前 App；`esc` 关闭。
- 直接敲字母/汉字：过滤词块（如敲 "13" 只剩含 13 的块）。
- `G`：粒度循环切换 字 → 词 → 智能短语（P1）。

### 3.5 反馈

- 复制成功：所选块聚拢飞向菜单栏图标 + HUD「已复制 ✓」；失败/空选择时 ⏎ 无操作并轻微抖动。
- 所有动画尊重系统「减弱动态效果」。

## 4. 技术架构

- **语言/框架**：Swift 6 + SwiftUI（App 入口 `MenuBarExtra`），AppKit 补位（NSPanel、NSEvent 全局热键、AX API）。纯原生，不引入任何跨平台/UI 框架依赖。
- **依赖最小化**：第三方仅 GRDB（原生 SQLite 封装）一个；热键、玻璃、分词、实体识别全用系统框架。包体目标 < 10 MB。
- **存储**：GRDB（SQLite + FTS5）。选 GRDB 而非 SwiftData：分页/FTS/迁移可控性更好，符合懒加载策略。
- **模块划分**：

```
ClipstrateApp
├── Capture    ClipboardMonitor（changeCount 轮询 0.3s）
│              PrivacyGate（accessBehavior 检测与引导 / Concealed 过滤）
├── Store      HistoryStore（GRDB，分页 + FTS5）
│              BlobStore（图片/大文本落盘，LRU 清理）
├── Chop       Segmenter 协议 ← NLSegmenter（NLTokenizer 实现，保留换引擎接缝）
│              EntityDetector（NSDataDetector + RegexEntity 规则表）
├── UI         MainPanel（历史列表，懒加载）
│              ChopPanel（拆词面板）· EntityHUD（复制即拆胶囊）
│              Onboarding（权限引导）· Settings
└── System     HotkeyManager · SelectionGrabber（AX + ⌘C 回退）· PasteService
```

- 实体规则表数据驱动（正则 + 校验函数 + 图标 + 优先级），P2 开放用户自定义即基于此表。
- **兼容性接缝（不堵死向下兼容）**：所有 macOS 26-only API 集中在三处，业务代码里不出现 `#available` 判断：
  1. `GlassSurface` 统一样式修饰符——内部用 `.glassEffect()`，日后加 `.ultraThinMaterial` / `NSVisualEffectView` 分支即可跑在 macOS 15/16；
  2. `PrivacyGate`——`accessBehavior`、`detect` 等新隐私 API 的探测与引导全部隔离于此，旧系统直接短路为"无需授权"；
  3. 玻璃形变（morphing）动画的降级开关——旧系统退化为普通淡入淡出。
  其余核心依赖（NLTokenizer、NSDataDetector、GRDB、MenuBarExtra、AX API）本身支持 macOS 13+，无兼容负担；arm64 单架构只是构建开关，需要 Intel 时切 universal 即可，不构成架构约束。

## 5. 性能与内存工程（硬约束）

原生、快、省是产品底线，不接受为功能牺牲流畅度。所有预算写成可自动验收的数字，超标按 bug 处理。

### 5.1 性能预算

| 指标 | 预算 | 测量方式 |
|---|---|---|
| 面板唤出（按键 → 上屏） | < 100 ms（P95），体感一帧到位 | `os_signpost` 打点 + Instruments |
| 搜索响应（每次击键） | < 30 ms | FTS5 后台查询，主线程只收结果 |
| 拆词（1000 字分词 + 实体识别） | < 30 ms（后台执行） | XCTest 性能用例 |
| 历史列表滚动（1 万条） | 120 Hz 不掉帧 | Instruments Hangs / Core Animation |
| 常驻内存（空闲挂机 24h） | < 30 MB，目标 25 MB | 定时采样 footprint，曲线必须平直 |
| 峰值内存（面板 + 大图预览） | < 100 MB，关闭后回落基线 | 内存基准脚本 |
| 冷启动（登录项 → 开始监听） | < 300 ms | `os_signpost` |
| 包体 | < 10 MB | 无大依赖、无内置词典 |

### 5.2 不卡顿的结构性保证

- **主线程只做 UI**：分词、正则、DB 读写、图片解码全部在后台 actor；Swift 6 strict concurrency 让越线在编译期报错。
- **面板预热**：启动即创建隐藏 NSPanel + Hosting 视图树，唤出只做数据绑定 + `orderFront`——这是 <100ms 的关键，代价仅几 MB 常驻，值得。
- **轮询低成本**：`DispatchSourceTimer`（0.3s + 100ms leeway，允许系统合并唤醒省电），回调只比较 `changeCount` 整数，变化才真正读内容；每轮包 `autoreleasepool`。
- **图片永不全解码**：ImageIO `CGImageSourceCreateThumbnailAtIndex` 直接降采样生成缩略图落盘；UI 层用限额 `NSCache` 缓存缩略图，原图只在预览时按需解码。
- **DB**：WAL 模式、keyset 分页、FTS5 索引；打开面板只取首屏一页。

### 5.3 零泄露纪律

- 常见泄露点清单（写进 `AGENTS.md`，AI 生成代码时逐条对照）：
  - NSPanel 设 `isReleasedWhenClosed = false` 且只有单一持有者；
  - 全局事件监听（`NSEvent.addGlobalMonitor`）、通知 observer、`DispatchSource` 必须成对移除/cancel；
  - timer、异步回调、通知闭包捕获一律显式 `[weak self]`。
- **自动化验收**：脚本执行"开关面板 ×100 + 连续复制 1000 条"后，footprint 必须回到基线 ±5%，Instruments Leaks 零检出。
- **每个里程碑跑三件套**：Leaks / Allocations / Hangs，外加 Thread Performance Checker。
- **长期驻留验证**：发布前连续挂机 24h，采样内存曲线无爬升。

## 6. 里程碑

| 阶段 | 内容 | 验收标准 |
|---|---|---|
| **M0 骨架**（~3 天） | MenuBarExtra + 轮询采集 + GRDB 入库 + 权限 onboarding | 复制的内容出现在菜单栏列表；隐私弹窗只出现一次 |
| **M1 主面板**（~1 周） | ⌥V 玻璃面板、搜索、回车粘贴、图片缩略图 | 日常可替代 iCopy 的历史功能 |
| **M2 拆词**（~1.5 周） | ChopPanel 三入口、NLTokenizer 分词、P0 实体、键盘/鼠标双流 | "⌘C → ⌥X → 1 复制淘口令" 跑通 |
| **M3 打磨**（~1 周） | 炸开/聚拢动画、性能与泄露验收（§5 全部预算）、边界（超长文本/巨图/密码过滤） | §5.1 预算全部达标；开关面板 ×100 内存回基线、Leaks 零检出 |
| **M4 发布** | 图标、官网页、公证签名分发（暂不上 MAS，避免沙盒限制 AX 权限） | 可分发 dmg |

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 剪贴板隐私弹窗（macOS 15.4+/26） | 不设"始终允许"则轮询反复弹窗，产品不可用 | Onboarding 第一屏专门处理；用 `detect` API 减少真实读取次数 |
| AX 取词在 Electron/部分浏览器失效 | 入口 C 不稳定 | 自动回退模拟 ⌘C（保存并恢复剪贴板现场）；入口 A/B 不受影响 |
| NLTokenizer 对新词/口令切分不佳 | 拆词观感差 | 实体层优先兜底；标点弱化；P1 粒度切换；`Segmenter` 协议保留换引擎余地 |
| 淘口令格式频繁变化 | 识别失效 | 规则表数据驱动、宽松匹配 + 长度校验，更新只改配置 |
| MAS 沙盒限制 | 无法上架完整功能 | 首发走官网公证分发，MAS 版可后置裁剪 |
