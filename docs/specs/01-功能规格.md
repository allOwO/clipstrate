# 01 · 功能规格

> 精确到可直接实现的行为定义。UI 尺寸/动效以 `prototype/index.html`（变体 C + 通铺设置）为像素参考。
> 优先级标注：〔P0〕MVP 必须；〔P1〕第一次迭代；〔P2〕观察后再做。未标注默认 P0。

## 1. 剪贴板采集

### 1.1 监听
- `DispatchSourceTimer` 轮询 `NSPasteboard.general.changeCount`，间隔 300ms、leeway 100ms；计数不变直接返回，变化才读内容；每轮包 `autoreleasepool`。
- 读取顺序（取第一个命中的主类型）：
  1. `fileURL`（一个或多个）→ kind = file
  2. `png` / `tiff` → kind = image
  3. `rtf` / `html` + `string` → kind = text（is_rich = true）
  4. `string` → kind = text
- **跳过不入库**：
  - 含 `org.nspasteboard.ConcealedType` 或 `org.nspasteboard.TransientType`（密码管理器等）；
  - 含自写入标记 `io.github.allowo.clipstrate.self`（本 App 粘贴时自己写的）；
  - 空字符串 / 纯空白；
  - 单条超限：文本 plain > 2 MB 截断存储并标记 truncated；图片原始数据 > 50 MB 丢弃并 HUD 提示。

### 1.2 富文本（双份存储）
- is_rich 条目**同时保存**：富文本原始数据（RTF 优先，其次 HTML，落盘 blob）+ 纯文本副本（`plain_text` 列）。
- 纯文本副本用于：搜索（FTS）、去重 hash、分词、「纯文本」粘贴。
- 粘贴默认写回富文本原始类型 + 纯文本两种 representation；「纯文本」动作只写 `string`。

### 1.3 去重与排序
- `content_hash` = SHA-256(kind + 规范化内容)：text 用 plain_text；image 用原始数据；file 用排序后的路径列表。
- 命中已有 hash：不新建行，更新 `last_used_at` 并置顶；`pinned` 状态保留。
- 列表按 `last_used_at DESC`，置顶项（pinned）恒在最前〔P1〕。

### 1.4 来源 App
- 两级获取：① 剪贴板上的 `org.nspasteboard.source`（bundle id，最可信）→ ② changeCount 变化当拍的 `NSWorkspace.shared.frontmostApplication`。
- 存 `app_bundle_id` + `app_name`；图标运行时经 `NSWorkspace.shared.icon(forFile: appURL)` 取，**不落库**。
- 两级都失败 → 存 NULL，**UI 整个不显示来源角标**（不猜、不显示占位图标）。

### 1.5 隐私门（PrivacyGate，接缝之一）
- 启动时检查 `NSPasteboard.general.accessBehavior`：非 `.alwaysAllow` 时进入 Onboarding 引导（见 §8）；
- 可用时先用 `detect` 系 API 探测类型，减少真实读取次数；
- 旧系统（未来兼容模式）此门直接短路为"允许"。

## 2. 存储与配额

- GRDB（SQLite，WAL）；Schema 见 02 §4。图片原图 + 富文本 blob 落盘 `Application Support/Clipstrate/blobs/`（文件名 = content_hash），数据库只存相对路径。
- 图片缩略图：ImageIO `CGImageSourceCreateThumbnailAtIndex` 生成 maxPixel 512 的 JPEG 落盘 `thumbs/`；UI 只加载缩略图，原图仅 QuickLook 式预览时按需解码。
- **配额清理（RetentionJanitor）**：每次启动 + 每小时执行：
  1. 删除超过存储时限（设置：天/周/月/季度/半年/年/无限制，默认 月）的未置顶条目；
  2. 若 blobs+db 总量仍超磁盘上限（设置：256MB/512MB/1GB/2GB，默认 512MB），按 `last_used_at` 从旧到新删未置顶条目直至达标；
  3. 删行时同步删除对应 blob/thumb 文件（事务后）。

## 3. 唤出面板（SummonPanel）——核心 UI

### 3.1 唤出与定位
- 触发：全局快捷键（默认 `⌥V`，可改）；〔P1〕双击修饰键、Fn 单击。
- 位置：优先文本光标——AX 取焦点元素 `kAXSelectedTextRange` + `kAXBoundsForRangeParameterizedAttribute` 得插入点屏幕矩形；失败退鼠标位置。卡片条**底边**位于光标/鼠标上方 12pt，水平居中于锚点，整体 clamp 进当前屏幕 visibleFrame。
- 窗口：无边框非激活 NSPanel（细节 02 §6），**不抢当前 App 焦点**。
- 再次按快捷键 / `esc` / 点击面板外 → 关闭。

### 3.2 形态（原型变体 C）
- **无外层容器**：每张卡片是独立玻璃岛（GlassSurface，圆角 20），底边对齐一条基线；卡片间距 10。
- 默认显示最近 **9** 条（对应数字键 1–9），横向排布；超出屏宽时条带可横向滚动，选中项自动滚入视野。
- 未选中卡 128×126；选中卡 252×196（向上生长，基线不动），过渡 0.28s spring。
- 卡片内容：
  - 顶行：类型标签（文本/富文本/图片/文件）+ 来源 App 图标名（可缺省）+ 右上角序号角标 1–9；
  - 文本卡：正文摘要（未选中 4 行 / 选中 6 行，富文本带格式渲染）；
  - 图片卡：缩略图 + 尺寸/格式/大小 meta；文件卡：图标 + 文件名 + 路径 meta；
  - 选中的**文本卡**左下角两个胶囊按钮：`纯文本`、`✂︎ 分词`（图片/文件卡无按钮）。
- 快捷键提示：独立悬浮玻璃胶囊，居中于卡片条下方 18pt：`←→ 选择 · ↓ 动作 · ⏎/点击选中卡 粘贴 · ⌥⏎ 纯文本粘贴 · Tab 分词 · esc 关闭`。
- 进场：卡片自基线上浮 26pt+scale 0.92→1 逐张错开 45ms（仅唤出时播一次）；关闭反向 0.15s。尊重"减弱动态效果"（接缝三：降级为淡入淡出）。

### 3.3 键盘（两层焦点）
| 键 | 卡片焦点层 | 动作焦点层 |
|---|---|---|
| `←` `→` | 切换选中卡（循环），重置动作焦点 | 同左（先回卡片层再切） |
| `↓` / `↑` | 进入动作层：↓ 依次 纯文本→分词→回卡片（循环），↑ 反向；图片/文件卡无动作层，忽略 | 同左循环 |
| `⏎` | 粘贴选中项 | 执行焦点按钮 |
| `⌥⏎` | 纯文本粘贴（文本卡） | — |
| `Tab` | 打开分词层（文本卡） | 同左 |
| `1–9` | 直接粘贴第 N 条（须在设置开启数字快捷键修饰键时为 修饰键+N；设置为"暂无"时面板内裸数字生效） | — |
| `esc` | 关闭面板 | 回卡片焦点层 |
- **焦点视觉**：卡片层 = 选中卡 2.5pt accent 描边；动作层 = 焦点按钮 accent 描边+浅底，同时**选中卡描边淡化至 28% 透明度**（0.2s 过渡）。

### 3.4 鼠标
- 点击未选中卡 = 选中；**点击已选中卡 = 粘贴**；hover 卡片轻微上浮 4pt。
- 按钮点击即执行。无右键菜单（P1 考虑：置顶/删除）。

### 3.5 粘贴行为（PasteService）
1. 把条目写入 `NSPasteboard.general`（富文本写 RTF+string；纯文本动作只写 string；图片写原图数据；文件写 fileURL），附加自写入标记类型；
2. 关闭面板，把键盘焦点交还原 App；
3. CGEvent 合成 `⌘V` 发往前台 App（需辅助功能权限）；未授权则只完成复制并 HUD 提示「已复制，请 ⌘V 粘贴」。
- 设置项"按下后 / 双击回车"= 输入剪贴板内容（默认）或 仅复制（跳过第 3 步）。

### 3.6 面板内搜索（type-to-search）

- **无常驻输入框**（保持无框卡片的干净）：面板打开后直接敲可打印字符即进入搜索态；卡片条上方 12pt 浮出**玻璃搜索胶囊**（🔍 + 查询词 + 匹配数），实时过滤卡片（防抖 100ms，后台 FTS）。
- 过滤结果就地重排（仍最多 9 张，序号 1–9 对应过滤后顺序）；无结果显示「无匹配」空态胶囊。
- 键位调整：搜索态下**裸数字键并入查询词**（数字直贴仅在查询为空时生效）；`⌫` 删字符；`esc` 第一次清空查询回全量、第二次关面板；`←→↓↑⏎ Tab` 导航键行为不变。
- **中文输入**：面板唤出后即临时成为 key window，并预聚焦一个不可见的文本输入客户端；中英文均可从第一个字符直接键入，**无需先按 `/`**。空查询仍不显示搜索胶囊；输入法组合期间按键交给 IME，提交后实时过滤。`esc`/失焦自动退出。
- **搜索范围（面板与 Popover 一致）**：

| 条目类型 | 匹配字段 |
|---|---|
| 文本/富文本 | 正文 `plain_text`（FTS5 trigram，中文子串可搜） |
| 文件 | 文件名（`label`，多文件拼接） |
| 图片 | `label`（如「截图 2026-07-21」）；〔P2〕Vision OCR 文字入索引后可搜图中文字 |
| 全部类型 | 来源 App 名（`app_name`） |

## 4. 分词层（ChopOverlay）

### 4.1 入口
- A〔P0〕：面板内选中文本卡 → `Tab` / `↓` 到「✂︎分词」→ `⏎` / 点按钮；
- B〔P0〕：**EntityHUD**——复制的文本检测到实体时，屏幕右上角浮出胶囊（图标+「N 个实体 · ⌥X 拆词」），2.5s 自动消失；点击或全局 `⌥X` 展开分词层；
- C〔P1〕：任意 App 选中文本按 `⌥X` 直接拆词（SelectionGrabber：AX `kAXSelectedText` → 失败则模拟 ⌘C 读取后**恢复剪贴板现场**）。

### 4.2 布局（玻璃容器面板，宽 ≤640）
- 顶部「智能识别 · 按数字键直接复制」标题 + 实体行：每个实体 = 数字徽章(1–9) + 图标 + 值 + 类型标签的胶囊；**无实体时整行隐藏**；
- 中部词块流：保持原文顺序 flow 排布，标点词块 38% 透明度弱化；分词器见 02 §3；
- 底部状态栏：`已选 N 词 · 点按选词 · 按住划选一段 · ⏎ 复制所选 · ⇧⏎ 复制并粘贴 · esc 返回`。
- 打开时背后的唤出面板透明度降至 25%；esc 返回恢复。

### 4.3 交互
- **点按**词块：切换选中（accent 填充）；
- **按住划选**：mousedown 切换锚点词块并确定模式（选/反选），按住划过时对 [锚点, 当前] 闭区间应用该模式、区间外恢复按下前快照（往回划收缩），mouseup 结束；
- `1–9`：复制对应实体 → toast「已复制：值 ✓」→ 关闭整个面板；点击实体胶囊同效；
- `⏎`：把选中词块**按原文顺序**拼接复制（默认无连接符；设置可选空格/换行〔P1〕）→ 关闭；空选择时 `⏎` 无操作 + 面板轻微抖动；
- `⇧⏎`：复制并粘贴（走 PasteService）；
- `⌘A` 全选词块；`esc` 返回面板。

### 4.4 实体规则（EntityDetector）
- 两级：`NSDataDetector`（URL、电话、日期、地址）+ 自定义正则表（数据驱动：id/正则/校验函数/图标/优先级/是否 P0）。
- P0 规则：

| 实体 | 模式（示意，实现以测试集为准） | 校验 |
|---|---|---|
| 邮箱 | `[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}` | — |
| 中国手机号 | `(?<!\d)1[3-9]\d{9}(?!\d)` | — |
| URL | NSDataDetector .link | — |
| 淘口令 | `([￥$€£¢₳¤])\s*([0-9A-Za-z]{8,14})\s*\1` 及 `\(\([0-9A-Za-z ]{8,20}\/\/` 变体 | 中段长度 8–14 |
| 验证码 | `(?:验证码|校验码|动态码|code)\D{0,6}(\d{4,8})`；上下文缺失时独立 4–6 位数字降权 | — |

## 5. 菜单栏与 Popover

- `NSStatusItem` 常驻图标（模板图标 ✂︎ 风格）；**左键单击直接弹 Popover 主界面，不显示菜单**；右键（或 ⌃点击）弹最小菜单：`设置… / 关于 / 退出`（退出的唯一入口，必须有）。
- Popover：锚定图标下方的玻璃面板（宽 620，横向布局）：
  - 头部：App 名 + 搜索框（常驻输入框，输入即搜，防抖 150ms；范围同 §3.6 表：正文/文件名/图片 label/来源 App）；
  - 中部：横向卡片流（懒加载分页，滚到末尾加载下一页 50 条）；**点击条目 = 复制到剪贴板顶部** + toast；hover 显示删除 ✕〔P1〕；
  - 底部：`共 N 条 · X MB` 统计 + `⚙️ 设置` `ⓘ 关于` 按钮。
- 点击面板外自动关闭。

## 7. 数据备份

### 7.1 导入 / 导出〔P1，M4 实现〕
- 导出：勾选 配置信息 / 忽略名单 / 剪贴板历史数据库（默认全选）→ NSSavePanel 存单文件 `Clipstrate-backup-yyyyMMdd.clipstrate`。
- `.clipstrate` = zip：`manifest.json`（format_version、app_version、导出时间、包含项）+ `settings.json` + `ignore.json` + `history.sqlite` + `blobs/` + `thumbs/`。
- 导入：NSOpenPanel 选文件 → 校验 manifest 版本 → 确认弹窗（历史导入 = **合并**，hash 去重；设置导入 = 覆盖）→ 执行 + 结果 toast。

### 7.2 iCloud 云备份〔P1，M4 实现〕

**通道决策**：CloudKit / ubiquity container 需要付费 Apple Developer Program 提供的 iCloud capability；v0.1 **自动写入用户 iCloud Drive 文件夹** `~/Library/Mobile Documents/com~apple~CloudDocs/Clipstrate/`（系统自动同步、不向用户索要目录）。架构上隔离为 `BackupTransport` 协议（`CloudDocsTransport` 实现），未来加入付费开发者计划后可替换为应用专属容器。

- **格式**：完全复用 §7.1 的 `.clipstrate` 包；文件名 `Clipstrate-<设备名>-yyyyMMdd-HHmm.clipstrate`（多设备天然不冲突，备份是单向快照、不做合并同步）。
- **DB 快照**：用 SQLite `VACUUM INTO` 导出紧凑副本（WAL 模式下安全、不锁写），不直接拷贝在用的 db 文件。
- **自动备份**（`backup.autoICloud` 开时）：内容变更后 debounce 5 分钟触发；含历史库的全量包**每日至多一次**，纯设置/忽略名单的小包（KB 级）变更即传；内容 hash 无变化则跳过。
- **保留策略**：目录内保留最近 3 份 + 每周 1 份（最多 8 份），超出删最旧。
- **恢复**：列出目录内备份（含其他设备的）→ 选择 → 确认弹窗 → 恢复前先本地自动快照一份 → 合并导入（同 §7.1 语义）。文件被 iCloud 驱逐成 `.icloud` 占位时，`FileManager.startDownloadingUbiquitousItem` 触发下载并显示进度。
- **不可用态**：未开启 iCloud Drive（CloudDocs 目录不存在）→ 分区按钮置灰 + 「请在系统设置开启 iCloud 云盘」引导。
- 状态行：备份状态（已备份/待上传/不可用）、上次备份时间（读目录内最新包的 manifest）。

### 7.3 忽略名单〔P1〕
- 设置·交互区追加 App 列表：在名单内的 App 复制的内容不入库（按 bundle id 匹配前台应用）。

## 8. Onboarding（首启，两步）

1. **剪贴板权限**：说明 + 触发一次读取让系统弹「允许访问剪贴板」→ 引导选「始终允许」；轮询 `accessBehavior` 显示 ✓；被拒则给出系统设置深链。
2. **辅助功能**：说明（用于自动粘贴 ⌘V、光标定位、划词）→ `AXIsProcessTrustedWithOptions(prompt)` → 打开系统设置面板；轮询 `AXIsProcessTrusted()` 显示 ✓。可跳过（功能降级：只复制不粘贴、面板出现在鼠标处）。
- 完成后写 `onboarding.done`；后续启动检测权限缺失时菜单栏图标标黄点 + Popover 顶部提示条。

## 9. 反馈

- Toast/HUD：玻璃胶囊，底部居中，1.4s 自动消失；文案见各处规格。
- 复制成功动画：选中词块/卡片聚拢飞向菜单栏图标（0.3s，减弱动态时禁用）〔P1 打磨项〕。

## 10. 剪贴板堆栈〔P1〕

- `⌃⇧C` 开启/关闭堆栈模式：开启时菜单栏图标变体提示；每次复制入栈（正常入历史）；`⌃⇧V` 弹出栈顶并粘贴（FIFO，可配 LIFO？——不做，固定 FIFO）；栈空时提示音+HUD；关闭模式清空栈。
