# 02 · 架构与选型

## 1. 技术选型（含理由与备选）

| 领域 | 选型 | 理由 | 落选备选 |
|---|---|---|---|
| 语言/UI | Swift 6（strict concurrency）+ SwiftUI，AppKit 补位 | 原生性能与观感；并发越线编译期报错 | Electron/Tauri（内存/观感不达标） |
| 部署目标 | macOS 26+，arm64 单架构 | 全量 `glassEffect`；universal 只是构建开关 | 兼容旧系统（接缝已留，后置） |
| 存储 | **GRDB.swift 7.x**（SQLite，WAL） | 分页/FTS5/迁移完全可控，配合懒加载 | SwiftData（渲染全量取行的前车之鉴）、CoreData（样板重） |
| 全文搜索 | FTS5 `tokenize='trigram'` | 中文子串搜索开箱即用（系统 SQLite ≥3.34） | jieba 索引（依赖重）、LIKE（慢） |
| 分词 | NaturalLanguage `NLTokenizer(unit: .word)` | 离线、零依赖、中英混排可用；不足由实体层兜底 | jieba 移植（词典数十 MB）、在线 API（隐私否决） |
| 实体识别 | `NSDataDetector` + 自定义正则规则表 | 数据驱动，P2 可开放自定义 | — |
| 全局热键 | **KeyboardShortcuts**（sindresorhus） | 录制控件+注册一体，成熟轻量 | Carbon RegisterEventHotKey 手写（录制 UI 成本高） |
| 登录启动 | `SMAppService.mainApp` | 系统标准 | LaunchAgent plist（过时） |
| 图片缩略图 | ImageIO `CGImageSourceCreateThumbnailAtIndex` | 降采样解码，不全图进内存 | NSImage resize（先全解码，否决） |
| 工程生成 | **XcodeGen**（`project.yml`） | AI/人都易编辑，diff 干净 | 手管 xcodeproj（合并地狱）、Tuist（过重） |
| 分发 | 官网 dmg + 公证（非 MAS） | 沙盒会限制 AX/CGEvent | MAS（后置裁剪版） |
| iCloud 备份通道 | 直写用户 iCloud Drive 文件夹（`~/Library/Mobile Documents/com~apple~CloudDocs/Clipstrate/`），`BackupTransport` 协议隔离 | iCloud entitlement 仅限 MAS，Developer ID 不可用；CloudDocs 免 entitlement、系统自动同步 | CloudKit / ubiquity container（MAS 版再换，接缝已留） |
| DB 快照 | SQLite `VACUUM INTO` | WAL 下安全导出紧凑副本，不锁写 | 直接拷 db 文件（WAL 下可能不一致） |

**依赖白名单 = GRDB.swift + KeyboardShortcuts，共两个。** 新增依赖需用户批准。

## 2. 模块划分

```
Clipstrate/
├── App/            ClipstrateApp（@main）、AppDelegate（NSStatusItem、生命周期）
├── Capture/        ClipboardMonitor（轮询 actor）
│                   PasteboardReader（类型解析→ClipItem 草稿）
│                   PrivacyGate（accessBehavior/detect，接缝②）
├── Store/          HistoryStore（GRDB DatabasePool；唯一 DB 入口）
│                   BlobStore（blobs/thumbs 落盘与删除）
│                   RetentionJanitor（配额清理）
│                   BackupService（.clipstrate 打包/合并导入、自动备份节流）
│                     └ BackupTransport 协议 ← CloudDocsTransport / LocalFileTransport
├── Chop/           Segmenter 协议 ← NLSegmenter
│                   EntityDetector + EntityRule 规则表
├── UI/             SummonPanel/（PanelController: NSPanel + SwiftUI 内容、定位、键盘）
│                   ChopOverlay/（分词层视图 + 划选手势）
│                   MenuBar/（StatusItemController、PopoverPanel）
│                   Settings/（通铺窗口 + scroll-spy）
│                   Onboarding/、EntityHUD、Toast
│                   DesignSystem/（GlassSurface 接缝①、动效降级接缝③、颜色/字体 token）
├── System/         HotkeyCenter（KeyboardShortcuts 封装）
│                   PasteService（写回剪贴板 + CGEvent ⌘V）
│                   SelectionGrabber（AX 取选中文本/光标 rect，⌘C 回退）
│                   LoginItem、SoundPlayer
└── Shared/         Settings（@AppStorage 封装，key 表见 §5）
                    ClipItem/ClipKind 等模型、Log（os.Logger + signpost）
```

依赖方向：`UI → Store/Chop/System → Shared`；Capture → Store；**禁止反向依赖**。模块间以 protocol 交互（如 `Segmenter`），便于测试与替换。

## 3. 关键流程

```
采集：Timer(300ms) → changeCount 变? → PrivacyGate.detect → PasteboardReader
     → 跳过规则(Concealed/自写入/超限) → hash 去重 → HistoryStore.upsert
     → BlobStore 落盘(富文本/原图/缩略图) → EntityDetector(文本) → 有实体? EntityHUD

唤出：⌥V → SelectionGrabber.caretRect(失败→鼠标) → PanelController.show(预热面板)
     → HistoryStore.page(0) 绑定 → 键盘/鼠标交互 → PasteService.paste(item)

分词：文本 → NLSegmenter.tokens(plain) + EntityDetector.entities(plain)
     →（后台并行，<30ms）→ ChopOverlay 渲染 → 选择 → 拼接复制/粘贴

备份：内容变更 → debounce 5min → 含历史库? 当日已传过则跳过
     → VACUUM INTO 快照 + settings/ignore 导出 → zip 成 .clipstrate
     → CloudDocsTransport 写入 + 清理旧份（留 3 近 + 1 周，≤8）
```

## 4. 数据库 Schema（GRDB migration v1）

```sql
CREATE TABLE item (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  kind          TEXT    NOT NULL,              -- 'text' | 'image' | 'file'
  is_rich       INTEGER NOT NULL DEFAULT 0,
  plain_text    TEXT,                          -- text 必有；搜索/分词/去重用
  label         TEXT,                          -- 可搜标签：file=文件名拼接；image=「截图 yyyy-MM-dd」等；〔P2〕并入 OCR 文字
  rich_type     TEXT,                          -- 'rtf' | 'html'
  blob_path     TEXT,                          -- 富文本原数据/图片原图（相对 blobs/）
  thumb_path    TEXT,                          -- 图片缩略图（相对 thumbs/）
  file_urls     TEXT,                          -- file: JSON 路径数组
  content_hash  TEXT    NOT NULL UNIQUE,
  app_bundle_id TEXT, app_name TEXT,
  byte_size     INTEGER NOT NULL DEFAULT 0,
  truncated     INTEGER NOT NULL DEFAULT 0,
  pinned        INTEGER NOT NULL DEFAULT 0,
  created_at    INTEGER NOT NULL,              -- Unix ms
  last_used_at  INTEGER NOT NULL
);
CREATE INDEX idx_item_recency ON item(pinned DESC, last_used_at DESC);

CREATE VIRTUAL TABLE item_fts USING fts5(
  plain_text, label, app_name, content='item', content_rowid='id', tokenize='trigram'
);
-- 同步触发器：item 增/删/改上述三列时维护 item_fts（GRDB migration 内建三个 trigger）
```

查询约定：列表一律 `LIMIT 50` keyset 分页（`WHERE (pinned,last_used_at,id) < (...)`）；搜索走 `item_fts MATCH`（三列同权匹配：正文/label/来源 App，覆盖 01 §3.6 范围表），面板防抖 100ms / Popover 150ms，后台队列执行。trigram 要求查询 ≥3 字符（中文 ≥3 字）；更短查询回退 `LIKE '%q%'`（短查询数据量小，可接受）。

## 6. 窗口策略

| 窗口 | 实现 | 要点 |
|---|---|---|
| 唤出面板 | `NSPanel(styleMask: [.borderless, .nonactivatingPanel])`，`level = .floating`，`isMovable = false`，`isReleasedWhenClosed = false`，重写 `canBecomeKey = true` | **启动即创建并常驻隐藏（预热）**，唤出仅数据绑定 + `orderFrontRegardless`；键盘走面板 keyDown/本地 monitor；`resignKey` / 点击外部即关 |
| 分词层 | 同一面板内的 SwiftUI 覆盖层（不是新窗口） | 背景卡片条降透明度 25% |
| EntityHUD / Toast | 复用一个小型非激活 NSPanel | 屏幕右上/底部居中，自动消失 |
| Popover | `NSPanel` 锚定 statusItem button 下方（非 NSPopover） | 玻璃自绘、无箭头；失焦即关 |
| 设置 | 标准 `NSWindow` + SwiftUI（通铺） | 单例，可调大小，记忆尺寸 |
| Onboarding | 标准窗口，居中 | 完成后释放 |

## 7. 并发模型

- **@MainActor**：全部 UI、PanelController、StatusItemController。
- **ClipboardMonitor**：自有串行 DispatchQueue 的 actor；只做 changeCount 比较与读取，产出 `ClipItem` 草稿后 `await HistoryStore`。
- **HistoryStore**：包装 GRDB `DatabasePool`（内部自带读写队列）；对外 async 接口；写操作串行、读并行。
- **Segmenter / EntityDetector / 缩略图生成**：`Task.detached(priority: .userInitiated)`；结果回主线程渲染。
- 规则：主线程禁止 DB/正则/解码；跨 actor 传值类型（`ClipItem` 为 `Sendable` struct）。

## 8. 兼容接缝（不堵死向下兼容，业务代码零 `#available`）

1. **GlassSurface**（DesignSystem）：`View` modifier，内部 `glassEffect()`；将来加 `.ultraThinMaterial`/`NSVisualEffectView` 分支即可跑 macOS 15/16；
2. **PrivacyGate**：`accessBehavior`/`detect` 全部隔离于此，旧系统短路；
3. **MotionPolicy**：炸开/聚拢/morphing 动画开关，旧系统或"减弱动态"降级为淡入淡出。

## 9. 性能与泄露工程（执行摘要，全文见 规划 §5）

- 预算：唤出 <100ms P95、击键搜索 <30ms、1000 字分词 <30ms、1 万条滚动 120Hz、常驻 <30MB、峰值 <100MB 回落、冷启 <300ms、包体 <10MB。
- `os_signpost` 打点位：`summon.show`、`search.keystroke`、`chop.tokenize`、`capture.tick`、`db.page`。
- 泄露闸门：开关面板 ×100 + 复制 1000 条后 footprint 回基线 ±5%、Leaks 零检出（脚本见 03 §T3.2）；NSPanel 单一持有者；监听器/DispatchSource 成对释放；异步闭包显式 `[weak self]`。
- 日志：`os.Logger(subsystem: "io.github.allowo.clipstrate", category: 模块名)`；Release 关 debug 级。

## 10. 错误处理约定

- DB/IO 错误：记日志 + 不崩溃降级（如缩略图失败→显示类型图标）；采集循环任何错误吞掉并记录，**绝不中断轮询**。
- 权限缺失：功能降级路径见 01 §8；不弹阻断性对话框。
- 断言只在 Debug（`assertionFailure`），Release 走日志。
